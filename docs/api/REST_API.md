# SuperSpace REST API Integration Guide

## Overview
SuperSpace provides a robust REST API to allow external web applications to seamlessly integrate with our mature workspace features (e.g., Studio/CMS Lite, CRM, Projects, Tasks). 

The API is served directly via our Convex real-time backend ensuring high-performance read/write operations with guaranteed strict RBAC compliance.

## Base URL
```text
https://<your-convex-deployment-url>.convex.site/api/v1
```

## Authentication
All API requests must include an `Authorization` header containing a valid API key scoped to a specific Workspace.

```http
Authorization: Bearer sk_live_...
```

*Note: API Keys can be generated by Workspace Administrators inside the Settings > API Keys menu.*

## Rate Limiting
- **Free Tier**: 100 requests / minute per IP.
- **Pro Tier**: 1000 requests / minute per IP.
If exceeded, the API will respond with `429 Too Many Requests`.

## Core Endpoints (v1)

### 1. CMS Lite (Studio)
Fetch public-facing content.

**List Posts**
- `GET /api/v1/cms/posts`
- **Query Params**: `locale=en`
- **Response**: Array of published blog posts.

**List Portfolio Items**
- `GET /api/v1/cms/portfolio`

### 2. CRM Integration
Sync external leads or contacts into your SuperSpace CRM pipeline.

**Create Lead**
- `POST /api/v1/crm/leads`
- **Body**:
  ```json
  {
    "name": "Jane Doe",
    "email": "jane@example.com",
    "source": "External Website Form"
  }
  ```

### 3. Projects & Tasks
Create support tickets or tasks directly from external applications.

**Create Task**
- `POST /api/v1/projects/tasks`
- **Body**:
  ```json
  {
    "title": "Fix login bug on mobile app",
    "description": "User reported...",
    "priority": "high",
    "assigneeId": "user_123"
  }
  ```

## Error Handling
The API returns standard HTTP status codes:
- `200 OK` - Success
- `400 Bad Request` - Invalid input (usually a Zod validation error)
- `401 Unauthorized` - Missing or invalid API Key
- `403 Forbidden` - API Key lacks permissions for this action
- `500 Internal Server Error` - Server-side issue

Example Error Response:
```json
{
  "success": false,
  "error": "Validation Error",
  "details": [
    { "field": "email", "message": "Invalid email address format" }
  ]
}
```
